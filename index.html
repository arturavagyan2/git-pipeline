🚀 Git Project Workflow

How to Work Like a Real Developer

For students learning team-based Git workflows

👩‍💻 1. Clone the Repository

git clone https://github.com/your-org/project.git

This gives you a local copy of the project

📦 2. Create a New Branch

git checkout -b feature/my-task

Always work on a separate branch. Never commit directly to main.

🔁 3. Stay Up to Date

Before pushing or merging, **pull the latest main**:


git checkout main
git pull origin main
git checkout feature/my-task
git merge main

Fix any conflicts if needed

🧪 4. Make Changes & Commit


git add .
git commit -m "Add feature X"

Commit often. Make each commit meaningful.

📤 5. Push Your Branch

git push origin feature/my-task

This publishes your work to GitHub

📬 6. Create a Pull Request (PR)

- Go to GitHub → New Pull Request

- Base: main

- Compare: feature/my-task

Ask for review. Never self-merge without checks!

⚠️ Common Mistakes

  • Working directly on main
  • Not syncing with main before PR
  • Messy or unrelated commits in PR
  • Forgetting to pull regularly

🧠 Why Not Just Pull?

💬 "Can’t I just do git pull on my branch?"

  • That pulls updates from your remote branch (e.g. origin/feature-x)
  • It doesn’t bring in changes from main
  • You’re still behind if main moved forward

🚫 Why git pull origin main Is Wrong Here

Doing this from your feature branch:

git pull origin main
  • It merges main into your branch in one step
  • ✅ But it hides the update to main in your local copy
  • ⚠️ You'll forget to update main next time
  • 🧼 Proper method is cleaner and separates concerns

✅ Proper Workflow Recap

  1. git checkout main
  2. git pull origin main ← 💡 updates local main
  3. git checkout feature-x
  4. git merge main ← brings latest main into your branch

🔁 Do this every time you want to work on your project

📊 Visualizing the Problem

Your branch is behind main


      main:   A──B──C──D
                     ↑
      feature: A──B──C──[Your work here]
  • main moved forward → Your branch is behind
  • You must pull + merge before continuing

✅ After Syncing

Proper sync brings you up to date:


      main:    A──B──C──D
                        ↑
      feature: A──B──C─────M
                           ↑
                     (merged main)
  • Your branch now includes all changes from main
  • Clean merge history, fewer conflicts

✅ How to Fix "Behind Main"

  1. git checkout main
  2. git pull origin main
  3. git checkout your-branch
  4. git merge main
  5. Fix conflicts & push

Now your branch is up to date.

🎯 Summary

  • ✅ One branch per person
  • ✅ Pull latest main often
  • ✅ PR only after syncing with main
  • ✅ Clean commits, meaningful messages
